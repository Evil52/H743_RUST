# Камера на WeAct MiniSTM32H7xx — Rust + Embassy

Живой видоискатель: сенсор **OV2640** (DVP) → **DCMI + DMA** → пинг-понг
буферы в AXI SRAM → даунсемпл 2:1 → **ST7735** 160×80 по SPI.
Асинхронная прошивка на [Embassy](https://embassy.dev) (executor + HAL),
без RTOS и без аллокатора.

```text
OV2640 --DVP--> DCMI+DMA --[FILLED]--> задача дисплея --> ST7735
                   ^                        |
                   +--------[FREE]----------+        + heartbeat-LED задача
```

## Структура (по мотивам «Making Embedded Systems» Elecia White)

| Слой | Файл | Что знает |
|------|------|-----------|
| Плата | `src/board.rs` | Клоки (HSE 25 МГц → 400 МГц), карта пинов, XCLK = MCO1/2 |
| Драйвер сенсора | `src/ov2640.rs` | Только SCCB/I²C. Generic над `embedded_hal_async::i2c::I2c` — переносится на любую плату |
| Вывод | `src/display.rs` | Панель, даунсемпл, «экран ошибок» |
| Приложение | `src/main.rs` | Задачи, каналы, порядок инициализации |

Идиомы из книги, применённые здесь:
- **Слоистость и абстракция железа** — драйвер сенсора не знает про STM32;
  всё платно-специфичное собрано в `board.rs`.
- **Zero-copy конвейер** — кадры не копируются: два буфера ходят по
  каналам `FREE`/`FILLED` как `&'static mut` (владение проверяет компилятор —
  одновременный доступ DMA и дисплея исключён типами).
- **Heartbeat** — отдельная задача мигает LED (PE3): «нет камеры» ≠ «прошивка умерла».
- **Видимые отказы** — если сенсор не отвечает/не тот ID, на LCD выводится
  `NO CAMERA`, в defmt-лог — причина.
- **Инструменты bring-up** — `set_colorbar()` включает тестовую заливку
  сенсора (закомментированный вызов в `camera_setup`).

Регистровые рецепты OV2640 перенесены из драйвера OpenMV
(`drivers/sensors/ov2640.c`, MIT) — та же прошивка, которую WeAct
официально шьёт на эту плату, т.е. таблицы проверены на этом железе.

## Пины (разъём DVP платы WeAct)

D0..D7 = PC6 PC7 PE0 PE1 PE4 PD3 PE5 PE6, VSYNC = PB7, HSYNC = PA4,
PIXCLK = PA6, XCLK = PA8 (MCO1, 12.5 МГц), SCCB = I2C1 (PB8/PB9).
LCD: SPI4 (SCK PE12, MOSI PE14), CS PE11, DC PE13, подсветка PE10.

## Сборка и прошивка

```bash
cd camera
cargo build --release

# Вариант 1: ST-Link + probe-rs (прошивка + defmt-логи в терминале)
cargo run --release

# Вариант 2: .bin для DFU / STM32CubeProgrammer
arm-none-eabi-objcopy -O binary \
  target/thumbv7em-none-eabihf/release/weact-h743-camera camera.bin
```

Если картинка есть, но цвета выглядят «перепутанными»: поменяйте
`rgb=false` на `true` в конструкторе `ST7735` (BGR/RGB панели) или
байтовый порядок пикселя в `display::draw_frame`.

## Ferrocene без подписки — как обстоят дела

[Ferrocene](https://ferrocene.dev) — это **тот же rustc**, зафиксированной
версии, с пакетом квалификационных документов (ISO 26262 / IEC 61508 /
IEC 62304). По состоянию на середину 2026:

- **Готовые бинарники** ставятся через `criticalup` и требуют платной
  подписки: Individual — €25/мес или €240/год. Бесплатного тира и
  триала нет.
- **Исходники полностью открыты** (MIT OR Apache-2.0, включая
  квалификационные документы): единственный легальный путь «без
  подписки» — собрать компилятор самому из
  [ferrocene/ferrocene](https://github.com/ferrocene/ferrocene)
  (`./x build` — это полная сборка rustc, несколько часов; результат
  функционально идентичен, но без подписи/сертификата).
- **Практический вывод для этого проекта**: обычный stable rustc
  генерирует тот же код — Ferrocene даёт только сертификационную
  бумагу. Наш таргет `thumbv7em-none-eabihf` входит в число
  квалифицированных таргетов Ferrocene, так что если сертификация
  когда-нибудь понадобится, проект соберётся без изменений.

## Возможные продолжения

- Кнопка K1 (PC13) → переключение colorbar / эффектов сенсора.
- JPEG-режим OV2640 (сенсор сжимает сам) + запись на TF-карту (SDMMC).
- Стриминг кадров по USB (CDC) на ПК.
