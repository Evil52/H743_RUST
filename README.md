# weact-h743 — RTC-часы с управляемым LED на STM32H743

Прошивка для отладочной платы **WeAct Studio MiniSTM32H7xx** (STM32H743VIT6),
написанная на чистом Rust без RTOS. Выводит текущее время и дату на встроенный
TFT-дисплей и управляет встроенным светодиодом через конечный автомат,
переключаемый кнопкой K1.

---

## 1. Аппаратная платформа

### 1.1. Почему WeAct STM32H743VIT6 Mini

| Параметр                | Значение                                     |
|-------------------------|----------------------------------------------|
| Ядро                    | Cortex-M7 @ 480 МГц (используется 240 МГц)   |
| Flash                   | 2 МБ                                         |
| RAM                     | 1 МБ (DTCM 128K + AXI 512K + SRAM1..4)       |
| FPU                     | двойной точности (FPv5-D16)                  |
| Кэши                    | I-cache 16 KB, D-cache 16 KB                 |
| USB                     | Type-C с прямой поддержкой DFU в ROM         |
| Дисплей                 | onboard 0.96" 80×160 TFT (ST7735) на SPI4    |
| Кнопки                  | RESET, BOOT0, K1 (PC13)                      |
| Светодиод               | LED1 на PE3 через NPN-транзистор             |
| Цена                    | ~$15 за плату                                |

Выбор обоснован:

- **Производительность без оверкила** — Cortex-M7 быстрее любого M0/M3/M4,
  но при этом доступнее по цене чем H7-Nucleo (~$25 за Nucleo-H743ZI2).
- **Встроенная периферия** — TFT, microSD, USB-C, флеш W25Qxx уже распаяны;
  не нужны дополнительные shield-платы.
- **DFU в ROM-bootloader** — заливка через USB по `dfu-util`, без отдельного
  программатора (ST-Link/J-Link опциональны).
- **Хорошо поддержана сообществом** — schematic V12 опубликован, есть готовый
  C SDK от WeActStudio. Pinout перенесён в нашу прошивку 1-в-1.

### 1.2. Pinout, использованный в прошивке

| Сигнал        | Пин   | Заметка                                       |
|---------------|-------|-----------------------------------------------|
| SPI4 SCK      | PE12  | Альтернативная функция AF5                    |
| SPI4 MOSI     | PE14  | Альтернативная функция AF5                    |
| LCD_CS        | PE11  | Удерживается LOW (единственное устройство)    |
| LCD_DC        | PE13  | Data/Command                                  |
| LCD_BL        | PE10  | Подсветка: **active-low** через P-MOSFET SI2301|
| LCD_RESET     | —     | Привязан к системному NRST на плате           |
| LED1          | PE3   | **Active-high** через NPN PDTC114E            |
| KEY K1        | PC13  | **Active-high**; внутренний pull-down         |
| VBAT          | PIN   | Опционально CR2032 для сохранения RTC         |

## 2. Почему Rust, а не C

| Критерий                  | C (HAL/CubeMX)                | Rust (`stm32h7xx-hal`)                       |
|---------------------------|-------------------------------|----------------------------------------------|
| Безопасность памяти       | Ручная; легко получить UB     | Гарантирована компилятором                   |
| Конкурентный доступ (ISR) | Volatile + ручная синхронизация| `Mutex<RefCell<…>>` + `Atomic*` — статически проверяется |
| Race conditions           | Возможны и часто незаметны    | Невозможны в safe Rust (нужен `unsafe`)      |
| Configuration ошибки      | Compile/runtime               | Type-state: ошибочные конфиги не собираются  |
| Размер прошивки           | ~30–50 KB для подобного функционала | ~40 KB (release, opt-level="s")        |
| Время старта проекта      | CubeMX генерирует код         | `cargo new` + 5 строк в Cargo.toml           |
| Управление зависимостями  | Submodule'ы / ручное копирование | `cargo` — единый менеджер                |
| Документация API          | Раскидана по PDF и хедерам    | `cargo doc` локально, rust-analyzer в IDE    |
| Macro-/code-gen           | Препроцессор C                | Декларативные и процедурные макросы          |

**Конкретные выгоды, реализованные в этой прошивке:**

1. **Type-state GPIO** — `gpioe.pe12.into_alternate::<5>()` возвращает
   `Pin<'E', 12, Alternate<5>>`. Перепутать AF-номер или использовать пин
   до конфигурации — ошибка компиляции, а не молчаливая поломка периферии.

2. **Безопасный shared state** — LED и таймер живут в
   `Mutex<RefCell<Option<…>>>`. В ISR доступ возможен только внутри
   `cortex_m::interrupt::free`, что атомарно отключает прерывания на время
   критической секции. В C это пришлось бы реализовывать руками с
   риском забыть `__disable_irq()`.

3. **Lock-free shared atomics** — `AtomicU8` для `MODE` и `AtomicU32` для
   `TICKS` позволяют main-loop читать состояние без блокировки ISR.

4. **Корректные wrapping-вычисления** — `wrapping_add`, `wrapping_sub`
   явно отделены от обычной арифметики; переполнение счётчика тиков не
   приведёт к неопределённому поведению.

5. **Нет heap** — `#![no_std]` + отсутствие `alloc`. Всё статически
   распределено; OOM невозможен по построению.

6. **Cargo вместо Makefile/CMakeLists.txt** — одна команда
   `cargo build --release` пересобирает всё, включая транзитивные зависимости
   (chrono, embedded-graphics, st7735-lcd).

## 3. Стек технологий

### 3.2. Crate-зависимости

| Crate                   | Версия | Роль                                       |
|-------------------------|--------|--------------------------------------------|
| `cortex-m`              | 0.7    | Доступ к core peripherals (NVIC, SysTick)  |
| `cortex-m-rt`           | 0.7    | Reset-handler, vector table, `#[entry]`    |

### 3.3. Архитектура прошивки

```
┌───────────────────────────────────────────────────────────────────┐
│                            main loop                              │
│  ┌──────────────────────────────────────────────────────────────┐ │
│  │  читает TICKS + MODE                                         │ │
│  │  если изменились → рисует время/дату/режим на ST7735         │ │
│  │  wfi() — спит до следующего прерывания                       │ │
│  └──────────────────────────────────────────────────────────────┘ │
└───────────────────────────────────────────────────────────────────┘
            ▲                                       ▲
            │ MODE/TICKS                            │ MODE
            │ (AtomicU8/U32)                        │ (AtomicU8)
            │                                       │
   ┌────────┴───────────┐                ┌──────────┴──────────┐
   │   TIM2 ISR (10Hz)  │                │    EXTI15_10 ISR    │
   │ ─ TICKS += 1       │                │ ─ дребезг через     │
   │ ─ управляет LED    │                │   DEBOUNCE_UNTIL    │
   │   по текущему MODE │                │ ─ MODE = MODE.next()│
   └────────────────────┘                └─────────────────────┘
```

**Прерывания (priority — дефолтный, одинаковый):**

- **TIM2** — периодический 100 мс. Тикает `TICKS`, рисует паттерн LED.
- **EXTI15_10** — фронт rising на PC13 (нажатие K1). Переключает `MODE`.

**Power management:** main loop вызывает `cortex_m::asm::wfi()` —
процессор останавливается до следующего прерывания, экономя ток.

### 3.4. State machine LED

```
   ┌─────┐  press   ┌──────┐  press   ┌──────┐  press   ┌───────┐  press
   │ OFF │ ───────▶ │ FAST │ ───────▶ │ SLOW │ ───────▶ │ SOLID │ ──────▶ OFF
   └─────┘          └──────┘          └──────┘          └───────┘
                    100 ms /          500 ms /          постоянно
                    100 ms            500 ms            горит
```

### 3.5. RTC и persistence

- Источник тактирования: **LSI** (внутренний RC ~32 кГц) — не требует
  внешнего кварца.
- Стартовое значение хранится в коде (`NaiveDate::from_ymd_opt(...)`).
- **Backup register 0** используется как magic flag `RTC_MAGIC = 0xCAFE_xxxx`.
  При совпадении магии — `set_date_time` пропускается, RTC продолжает
  идти с прошлого включения. При сбросе магии (новая прошивка с другим
  значением, или потеря питания backup-домена) — RTC seedится заново.
- Для сохранения времени **между полным отключением питания** требуется
  CR2032 на VBAT pad. Без батарейки RTC обнуляется при отключении USB.

## 4. Сборка и заливка

```bash
# Сборка release-бинаря
cargo build --release

# Конвертация ELF → raw binary
arm-none-eabi-objcopy -O binary \
  target/thumbv7em-none-eabihf/release/weact-h743 \
  firmware.bin

# Перевод платы в DFU: зажать BOOT0, нажать-отпустить RESET, отпустить BOOT0
# Заливка
dfu-util -a 0 -s 0x08000000:leave -D firmware.bin
```

После `:leave` плата автоматически выходит из DFU и запускает новую прошивку.
